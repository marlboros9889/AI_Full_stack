# 마린보이(MARINBOY) 미용실 예약 관리 서비스

## 프로젝트 소개

고객이 시술 메뉴와 가능한 시간을 확인해 예약하고, 관리자가 예약·휴무일·시술 메뉴를 통합 관리하는 웹 서비스입니다.

## 기술 스택

- Backend: Java 17, Spring Boot 3.5.3, Spring MVC, Spring Security, OAuth2
- Data: MyBatis, Oracle XE, XML Mapper
- Frontend: Thymeleaf, HTML, CSS, JavaScript
- Build/Test: Maven, Spring Boot Test, MyBatis Test

## 핵심 기능

- 고객: 회원/소셜 로그인, 시술 조회, 예약 가능 시간 조회, 예약 등록·변경·취소, 이용 내역 조회
- 관리자: 예약 목록·상세·상태·거절 관리, 휴무일 관리, 시술 메뉴·대표/갤러리 이미지 관리
- 공통: 세션 기반 권한 분리, 중복 예약 방지, 예외 응답 표준화, 개인정보 마스킹

## 담당 구현

- Controller-Service-DAO-MyBatis Mapper 계층 구조 설계
- 예약 가능 시간 및 중복 예약 검증 로직 구현
- 고객/관리자 권한별 API와 화면 흐름 구현
- Oracle 스키마·샘플 데이터·검증 SQL 작성
- 반응형 예약 및 관리자 UI 구현

## 문제 해결

| 문제 | 해결 | 결과 |
|---|---|---|
| 같은 시간대 중복 예약 가능성 | 서버에서 기존 예약과 시술 시간을 교차 검증 | 예약 데이터 정합성 확보 |
| 고객·관리자 기능 혼재 | 세션 역할 검사와 관리자 전용 API 분리 | 권한 없는 접근 차단 |
| 휴무일에도 예약 가능 | 휴무일과 영업시간을 예약 슬롯 계산에 반영 | 실제 운영 규칙 반영 |
| 이미지 관리 불편 | 메뉴별 대표·추가 이미지 업로드 지원 | 포트폴리오형 메뉴 노출 강화 |

## 아키텍처

`Browser → Spring Controller → Service → DAO → MyBatis XML → Oracle XE`

## 성과 및 회고

- 단순 CRUD를 넘어 예약 가능 시간, 권한, 휴무일 등 도메인 규칙을 서비스 계층에 구현했습니다.
- 화면과 API, 데이터베이스를 하나의 사용자 흐름으로 연결하며 풀스택 개발 과정을 경험했습니다.
- 향후 자동화 테스트 확대, 배포 환경 구성, 알림 연동으로 운영 완성도를 높일 계획입니다.
